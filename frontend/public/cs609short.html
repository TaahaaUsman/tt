Topic 1: Windows Operating System

The Windows Operating System is a special type of software that helps run your computer. It manages everything your computer doesâ€”like running programs, using memory, saving files, and connecting to devices like printers or USB drives.

Key Things Windows OS Does:

  1. Memory Management: Controls where programs are stored in memory and moves data to keep things running smoothly.
  2. File System Management: Helps save, open, and organize files in folders with long names.
  3. Processor Management: Shares work between multiple processors or cores to improve speed.
  4. Naming and Locating Resources: Gives special names to system parts like files and programs so it can find and use them.
  5. Multitasking: Lets you run many programs at once, like watching videos while typing.
  6. Communication and Synchronization: Helps programs share data safely without crashing.
  7. Security and Protection: Keeps your data safe and blocks illegal access.

Summary: Windows is like the manager of your computer. It handles memory, files, apps, security, and keeps everything working together.

Topic 2: Windows Evolution

The Windows Operating System has grown and changed a lot since it was first made. Each new version adds better features, looks nicer, and works faster. This process of improvement over time is called the evolution of Windows.

Why Windows Keeps Evolving:

  • Scalability: Works on all kinds of devicesâ€”PCs, servers, and mobiles.
  • Performance: Each version runs faster and more smoothly than older ones.
  • Integration: Supports modern tools like the web, media, and .NET platform.
  • Ease of Use: New versions have better menus, icons, and user interfaces.
  • Feature Upgrades: Adds useful tools and improves how the system works.

Main Versions in Windows History:

  • MS-DOS: Old text-only system used in the 1980s. No graphics.
  • Windows 3.1: First version with a graphical user interface (GUI).
  • Windows 95, 97, 98: Brought in better graphics and supported 32-bit systems.
  • Windows NT: Improved security and used the NTFS file system.
  • Windows XP, Vista, 7, 8, 10: Popular versions with strong features and performance.
  • Windows Server Editions: Used by companies to handle large-scale systems and networks.

Summary: From simple black-and-white screens to advanced, colorful, and secure systems, Windows has changed a lot. Each version is made to be faster, safer, and more user-friendly than the last.

Topic 3: File Systems & File Operations

A file system is the way an operating system stores and organizes files on a disk. The Windows OS supports many file systems, each with different features. File operations like creating, reading, writing, and deleting files are handled by special commands called APIs.

Main Windows File Systems:

  • NTFS: Modern file system. Supports large files, security, encryption, and recovery.
  • FAT12/16/32: Older systems. Simple but limited in file size and features.
  • exFAT: Good for USB drives. Handles large files but has less security than NTFS.
  • CDFS: Used for CD-ROMs.
  • UDF: Used for DVDs and Blu-ray discs.

File Naming Rules in Windows:

  • File names can be up to 255 characters long.
  • Cannot use characters like \ / : * ? " < > | .
  • Case-insensitive but case-preserving (e.g., file.txt = FILE.TXT).
  • Some names are reserved, like CON, PRN, NUL.

Common File Operations:

  • CreateFile: Opens or creates a file.
  • ReadFile / WriteFile: Reads from or writes to a file.
  • CloseHandle: Closes the file when done.

Standard C vs Windows API:

  • C functions like fopen() and fread() are cross-platform.
  • Windows APIs like CreateFile() offer more control and features.

Summary: Windows supports many file systems to handle different types of storage devices. File operations are done using system functions or standard C functions, depending on what you need.

Topic 4: Character Encoding & Generic Programming

In Windows programming, it's important to handle text properly in different languages. This is where character encoding comes in. Windows supports both old-style ASCII and modern Unicode. To write code that works with both, we use something called Generic Programming.

Character Encoding Types:

  • ASCII: Older format. Supports only English characters. Uses 1 byte per character.
  • Unicode (UTF-16): Supports all languages (Arabic, Chinese, Urdu, etc.). Uses 2 bytes per character.

API Variants:

  • Windows provides two versions of functions:
  • CreateFileA() â€“ for ASCII (A means ANSI)
  • CreateFileW() â€“ for Unicode (W means Wide)

Generic Programming Approach:

  • Use special types like TCHAR instead of char or wchar_t.
  • Use LPTSTR for strings that adapt to Unicode or ASCII.
  • Define macros like UNICODE and _UNICODE to switch between versions.
  • Use _tmain() instead of main() for portable main function.

Why Use Generic Code?

  • Same code can work in both Unicode and non-Unicode builds.
  • Makes it easier to maintain and upgrade.
  • Best practice for modern Windows development.

Summary: Windows supports both ASCII and Unicode. Using Generic Programming makes your code flexible, reusable, and future-ready for global applications.

Topic 5: Error Handling

In Windows programming, things can go wrongâ€”like a file not opening or a device not responding. Thatâ€™s why error handling is important. It helps you find out what went wrong and respond in a proper way.

Key Functions for Error Handling:

  • Display the message or log it for debugging.

Custom Error Reporting Function:

  • In large programs, itâ€™s common to write your own helper function like ReportError().
  • This function prints the user message and the system error.
  • Optionally, it can also exit the program if the error is serious.

Summary: Error handling in Windows uses functions like GetLastError() and FormatMessage() to help you understand and fix problems. Writing clean error reporting makes programs easier to debug and maintain.

Topic 6: Processes & Multitasking

Multitasking means running more than one program at the same time. In Windows, each running program is called a process. A process can do many things using smaller units called threads.

What is a Process?

  • A process is a program in execution (e.g., Chrome, Word).
  • Each process has its own memory and system resources.
  • Processes are created using APIs like CreateProcess().

What is a Thread?

  • A thread is a small unit of work inside a process.
  • A process can have one or many threads.
  • Threads share memory but run independently.

How Multitasking Works:

  • Windows OS switches between tasks quickly (CPU scheduling).
  • This switching happens so fast that it looks like tasks run at the same time.
  • Helps users play music, type, and download files all at once.

APIs and Functions:

  • CreateProcess(): Starts a new process.
  • ExitProcess(): Ends a process safely.
  • GetCurrentProcess(): Returns info about the current process.

Summary: Windows uses multitasking to run many apps at once. It manages processes and threads using special functions. Understanding these helps in writing efficient, responsive programs.

Topic 7: Interprocess Communication (IPC)

Sometimes different programs (processes) need to share data or send messages to each other. This is called Interprocess Communication, or IPC. Windows provides many safe and efficient ways to do IPC.

Why IPC is Needed:

  • Processes may work together to complete a task.
  • They may need to exchange files, messages, or signals.
  • Helps in building client-server and network-based applications.

Common IPC Methods in Windows:

  • Pipes: Allow one-way or two-way communication between processes.
  • Named Pipes: Used for communication between unrelated processes, even on different machines.
  • Anonymous Pipes: Used between related processes (like parent and child).
  • Shared Memory: Fastest method. Both processes access the same memory area.
  • Mailslots: For simple message-based communication.
  • Sockets: Used for communication over a network (e.g., internet).

Synchronization Tools:

  • Mutexes: Ensure only one process uses a resource at a time.
  • Semaphores: Limit how many processes access a resource.
  • Events: Let one process notify another when something happens.

Summary: IPC lets programs talk to each other and work together. Windows offers tools like pipes, shared memory, and sockets to safely share data and signals between processes.

Topic 8: Thread Management & Scheduling

A thread is the smallest part of a process that can run independently. Threads are useful for doing multiple tasks at the same time inside a single program. Windows lets you create and control threads, and decides which thread runs when â€” this is called scheduling.

What is a Thread?

  • It is a part of a process that does some work.
  • All threads of a process share the same memory and data.
  • Threads help make programs faster and more responsive.

Creating and Managing Threads:

  • CreateThread(): Starts a new thread.
  • ExitThread(): Stops a thread safely.
  • GetCurrentThread(): Gets info about the current thread.
  • Sleep(): Pauses the thread for a set time.

Thread Scheduling:

  • Windows decides which thread runs next using priorities.
  • Higher priority threads get more CPU time.
  • Threads can be paused and resumed using functions like SuspendThread() and ResumeThread().

Thread Synchronization:

  • WaitForSingleObject(): Waits for a thread or resource to finish or become available.
  • WaitForMultipleObjects(): Waits for more than one thread or resource.
  • These help avoid conflicts when threads try to access the same resource.

Summary: Threads allow a program to do many tasks at once. Windows gives control over creating, pausing, and ending threads. It also schedules them based on importance and keeps things running smoothly and safely.
